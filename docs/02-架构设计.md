# Agent 桌宠架构设计

## 1. 架构目标

项目采用 Clean Architecture，目标是让桌宠表现、对话编排、任务执行、记忆、持久化和平台能力彼此解耦。

核心规则：依赖只能由外向内。

```text
UI / Tauri / SQLite / LLM SDK
  -> infrastructure / interfaces
  -> application use cases
  -> domain entities and ports
```

## 2. 技术栈

详细选型依据、反向 challenge 和优化路线见 `08-技术选型挑战与优化.md`。本节只记录当前落地栈。

| 层面 | 技术 |
| --- | --- |
| 桌面容器 | Tauri 2 |
| 前端 | React + TypeScript + Vite |
| 状态 | Zustand |
| 动画 | Lottie light player + CSS fallback |
| 数据 | sql.js / SQLite 快照 |
| AI | OpenAI-compatible Gateway，后续可扩展 Anthropic/Gemini/DeepSeek |
| 测试 | Vitest、ESLint、smoke 脚本 |

## 3. 分层说明

| 层 | 目录 | 责任 | 禁止依赖 |
| --- | --- | --- | --- |
| Domain | `src/domain` | 实体、值对象、端口接口 | React、Tauri、SQLite、LLM SDK |
| Application | `src/application` | 用例、编排、DTO 规则 | UI 组件、Tauri API、数据库 SDK |
| Interfaces | `src/interfaces` | Controller、DTO、适配输入输出 | 具体数据库实现细节 |
| Infrastructure | `src/infrastructure` | LLM Provider、SQLite Repo、系统工具 | UI 组件 |
| UI | `src/ui` | 桌宠、聊天面板、设置面板 | 直接调用数据库/模型 SDK |
| App | `src/app` | composition root、依赖注入 | 业务逻辑堆叠 |
| Tauri | `src-tauri` | 窗口、权限、插件、系统能力 | 前端业务状态 |

## 5. 当前目录约定

```text
src/
  app/                 # Bootstrap and composition root
  application/         # Use cases by bounded context
  domain/              # Entities and ports
  infrastructure/      # LLM, SQLite, system tools
  interfaces/          # Controllers and DTOs
  shared/              # Errors, result types, logger
  styles/              # Global style
  ui/                  # React UI modules
src-tauri/             # Tauri Rust shell and permissions
scripts/               # Smoke and dev scripts
tests/                 # Unit and integration-style tests
docs/                  # Current canonical documentation
```

## 6. 关键端口

| 端口 | 所在层 | 实现 | 用途 |
| --- | --- | --- | --- |
| `LLMGateway` | domain | `LlmGatewayImpl` | 聊天、意图识别、任务文案、剪贴板处理 |
| `ToolExecutor` | domain | `TauriToolExecutor` / `WebToolExecutor` | 打开 URL/应用/文件夹、剪贴板、通知 |
| `ConversationRepository` | domain | SQLite Repository | 会话和消息保存 |
| `MemoryRepository` | domain | SQLite Repository | 用户记忆 |
| `UsageRepository` | domain | SQLite Repository | Token 和调用日志 |

## 7. 架构约束

- Domain 不得 import React、Tauri、SQLite、OpenAI SDK。
- Application 不得直接调用 `window.__TAURI__` 或具体插件。
- 每次 LLM 调用必须携带 `feature`，并尽量携带 `sessionId` 或 `taskId`。
- 每次有副作用的工具调用必须有输入校验、失败结果和用户可读反馈。
- 记忆写入必须显式触发，不能默认记录所有聊天内容。
- 人格提示词和任务执行逻辑分离，不能让角色文案决定安全策略。
- 新增 Provider 只能扩展 infrastructure，不应改动核心用例流程。

## 8. 当前架构风险

| 风险 | 影响 | 建议任务 |
| --- | --- | --- |
| Tauri dev 残留进程导致文件占用 | GNU check/build 失败、快捷键重复注册 | `BUG008`、`BUG009`、`BUG011` |
| `tsconfig` TypeScript 6 配置不匹配 | `npm run build` 失败 | `BUG012` |
| GUI 终验未完成 | MVP 不能标记 done | `TEST009` |
| usage 任务链路记录不完整 | 用量面板和成本统计不可信 | `DEV020`、`TEST008` |
| 本地密钥与快照安全未完全闭环 | 用户数据安全风险 | `BUG007` |
