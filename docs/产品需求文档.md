# Agent 桌宠产品方案与需求文档

## 1. 文档信息

- 产品名称：Agent 桌宠（暂定名：PetAgent）
- 默认角色形象：金毛犬
- 文档版本：v0.1
- 文档日期：2026-04-20
- 文档目标：明确产品定位、用户价值、MVP 范围、功能需求与技术实现方向，用于产品立项、原型设计与研发拆解。

## 2. 产品概述

Agent 桌宠是一款常驻桌面的智能陪伴与执行助手。它以“金毛犬桌面宠物”的拟人化形象存在，既能陪伴用户、提供情绪反馈，也能作为一个具备轻量任务处理能力的 Agent，帮助用户完成提醒、信息整理、快捷操作、知识问答与应用联动等工作。

产品核心不是“会动的桌面挂件”，而是“有个性、有记忆、可执行任务、低打扰”的桌面 AI 助手。用户在日常工作和生活中，可以像与宠物互动一样自然地与它沟通，同时获得真实可感的效率收益。

## 3. 产品愿景

打造一款让用户愿意长期留在桌面的 Agent 产品：

- 情感上，它是有性格、有反馈、有陪伴感的桌宠。
- 功能上，它是可理解上下文、能执行任务、能持续成长的桌面 Agent。
- 商业上，它可以从单机陪伴工具逐步升级为个人桌面入口与 Agent 平台。

## 4. 目标用户

### 4.1 核心用户

- 长时间使用电脑办公的知识工作者
- 喜欢桌面个性化、二次元/拟人化交互风格的年轻用户
- 希望通过 AI 提升轻任务处理效率的个人用户
- 需要提醒、陪伴与轻度情绪支持的独处型用户

### 4.2 典型用户画像

#### 用户 A：内容创作者

- 需要快速记录灵感、整理素材、获取写作建议
- 喜欢有趣、有生命感的产品，不愿意使用过于冰冷的 AI 工具

#### 用户 B：程序员/产品经理

- 长时间面对桌面，希望降低工具切换成本
- 需要快速问答、待办提醒、会议纪要整理、快捷打开工具

#### 用户 C：学生用户

- 需要学习提醒、番茄钟陪伴、知识问答与日常鼓励
- 对宠物养成、装扮、成长系统有较高兴趣

## 5. 用户痛点

- 传统 AI 助手入口重，打开频率低，难以成为常驻使用习惯
- 桌面宠物多为纯展示，缺乏真正可用的智能能力
- 用户处理碎片化任务时，经常需要切换应用，效率低
- AI 工具虽然强，但缺乏长期陪伴感、人格感和记忆连续性
- 提醒、待办、知识问答、快捷操作分散在多个工具中

## 6. 产品定位

### 6.1 产品一句话定义

一只住在桌面上的 AI 宠物，既陪伴你，也替你做一点事。

### 6.2 差异化定位

与通用聊天助手相比：

- 更轻量，常驻桌面，交互入口更自然
- 更拟人，具备角色设定、情绪表达与养成反馈
- 更场景化，聚焦桌面端高频微任务

与传统桌宠相比：

- 具备 LLM 对话、记忆和任务执行能力
- 可以调用本地工具、系统能力和第三方应用
- 能形成“陪伴 + 效率”的双重价值

## 7. 产品目标

### 7.1 业务目标

- 验证用户是否愿意长期将 Agent 桌宠常驻桌面
- 验证“陪伴感”与“任务执行”结合的产品价值
- 建立首批种子用户口碑与行为数据

### 7.2 用户目标

- 用户可在 1 分钟内完成安装与首次对话
- 用户可在 3 次互动内理解产品核心价值
- 用户可通过桌宠完成至少 3 类高频轻任务

### 7.3 MVP 成功指标

- 次日留存 > 35%
- 7 日留存 > 15%
- 人均日互动次数 > 8 次
- 首周至少 40% 用户使用过任务型能力
- 首周至少 30% 用户触发过主动提醒或陪伴反馈

## 8. 核心使用场景

### 8.1 陪伴场景

- 用户打开电脑后，桌宠主动打招呼
- 用户久坐或长时间未休息时，桌宠主动提醒
- 用户完成任务后，桌宠给予鼓励和反馈

### 8.2 助手场景

- 用户通过自然语言让桌宠创建待办、提醒事项
- 用户让桌宠总结剪贴板内容或当前文本
- 用户让桌宠快速打开指定应用/网页/文件夹

### 8.3 轻 Agent 场景

- 用户说“帮我整理今天会议待办”
- 用户说“每天下午 3 点提醒我喝水”
- 用户说“把这段内容总结成三点”
- 用户说“打开 Notion 并新建一篇会议记录模板”

## 9. 产品范围

### 9.1 MVP 范围

MVP 聚焦“桌宠形象 + 对话 + 记忆 + 本地轻任务执行 + 提醒系统”。

包含：

- 桌面宠物基础形象与状态动画
- 金毛犬角色设定与基础动作表现
- 聊天对话面板
- 基础人格设定与情绪反馈
- 用户基础资料与偏好记忆
- 待办、提醒、番茄钟
- 剪贴板总结/改写/翻译
- 打开应用、网页、文件夹等系统级快捷动作
- 基础设置中心

不包含：

- 多角色宠物商城
- 复杂工作流编排
- 多设备云同步
- 社区、分享、UGC
- 高风险系统操作
- 完整插件生态

### 9.2 后续扩展范围

- 多宠物角色与换装系统
- 更复杂的 Agent 工具调用与工作流
- 邮件、日历、文档平台接入
- 长期记忆与个性成长曲线
- 联网搜索与知识库
- 语音对话与情绪语音反馈
- 手机/平板/多端同步

## 10. 产品设计原则

- 低打扰：默认不频繁打断用户，主动行为可控
- 可爱但不幼稚：兼顾情感表达与办公场景适配
- 有用优先：每个“萌点”都尽量服务留存或使用频次
- 可信可控：所有执行动作可见、可确认、可撤销
- 渐进式智能：先做好高频轻任务，再逐步扩展 Agent 能力

## 11. 功能架构

### 11.1 一级模块

1. 桌宠形象系统
2. 对话与交互系统
3. Agent 执行系统
4. 记忆与个性系统
5. 提醒与陪伴系统
6. 设置与权限系统

### 11.2 二级模块说明

#### 1. 桌宠形象系统

- 常驻桌面显示
- 拖拽移动
- 状态动画：待机、开心、思考、忙碌、睡觉、提醒
- 点击反馈动作
- 气泡消息提示
- 金毛犬专属动作：摇尾巴、趴下、歪头、跑动、打哈欠

#### 2. 对话与交互系统

- 点击桌宠展开对话框
- 支持文本输入
- 快捷指令入口
- 历史消息查看
- 建议问题与引导语

#### 3. Agent 执行系统

- 意图识别
- 工具调用路由
- 执行动作确认
- 执行结果反馈
- 异常处理与兜底回复

#### 4. 记忆与个性系统

- 用户昵称、偏好、作息等基础记忆
- 近期上下文记忆
- 人格参数配置
- 陪伴关系值/亲密度

#### 5. 提醒与陪伴系统

- 定时提醒
- 待办列表
- 番茄钟
- 久坐提醒/喝水提醒
- 完成奖励反馈

#### 6. 设置与权限系统

- 开机自启动
- 始终置顶/边缘吸附
- 提醒频率设置
- 隐私开关
- 系统权限授权管理

## 12. 详细需求

## 12.1 桌宠基础能力

### 功能描述

桌宠以悬浮窗形式常驻桌面，支持显示、移动、点击交互与基础状态变化。

### 用户价值

- 建立产品存在感和陪伴感
- 降低打开 AI 工具的动作成本

### 需求点

- 支持 Windows 首发，后续可扩展 macOS
- 默认显示在桌面边缘，不遮挡主要工作区域
- 支持拖拽、吸边停靠
- 支持点击后展开聊天面板
- 支持不同状态动画切换
- 默认形象为金毛犬，主色调为金黄、奶油白、浅棕
- 表情和动作需体现亲近、可靠、活泼但不过度兴奋的金毛犬气质

### 验收标准

- 首次启动后 3 秒内完成桌宠渲染
- 拖拽过程流畅，无明显卡顿
- 点击桌宠 500ms 内弹出面板

## 12.2 对话系统

### 功能描述

用户可以通过自然语言与桌宠交流，完成闲聊、问答与任务发起。

### 需求点

- 支持普通闲聊
- 支持任务型命令输入
- 对高频任务给出快捷建议按钮
- 聊天框支持最近会话上下文
- 对话回复体现角色语气，但不影响信息清晰度

### 验收标准

- 常见输入可在 2 秒内返回首字
- 任务指令能够明确区分“聊天”和“执行”
- 回复风格统一、可识别

## 12.3 任务执行系统

### 功能描述

桌宠具备轻量工具调用能力，可执行低风险桌面任务。

### 首批支持任务

- 创建提醒
- 创建待办
- 开启番茄钟
- 总结/改写/翻译剪贴板内容
- 打开网页
- 打开本地应用
- 打开本地文件夹

### 执行规则

- 高风险动作不开放
- 涉及系统调用的操作先确认再执行
- 执行中显示状态反馈
- 执行完成后给出结果说明

### 验收标准

- 70% 以上预置意图可被正确路由
- 执行动作有明确确认和结果提示
- 失败场景有可理解的报错文案

## 12.4 记忆系统

### 功能描述

桌宠可以记住用户显式提供的信息和部分偏好，用于更自然的持续陪伴与任务服务。

### 可记忆内容

- 用户昵称
- 常用称呼
- 工作时间偏好
- 提醒偏好
- 常用应用
- 最近任务上下文

### 规则

- 所有长期记忆均可查看和删除
- 用户明确拒绝后不再自动记录对应内容
- 隐私信息默认不存储

### 验收标准

- 用户设置昵称后，下次启动仍可被正确称呼
- 设置的偏好可影响后续提醒策略
- 用户可一键清空记忆

## 12.5 提醒与陪伴系统

### 功能描述

通过轻提醒和情绪反馈，提升陪伴感与日常活跃度。

### 需求点

- 支持单次/重复提醒
- 支持番茄钟专注模式
- 支持喝水、休息、起身等健康提醒模板
- 支持任务完成时的情绪反馈
- 支持免打扰时段

### 验收标准

- 提醒触发误差不超过 10 秒
- 免打扰时段内不主动弹出打扰性消息
- 番茄钟流程完整闭环

## 12.6 设置中心

### 需求点

- 角色名称设置
- 语气风格强度设置
- 桌宠尺寸与透明度调整
- 开机自启开关
- 提醒开关和频率
- 记忆与隐私设置
- 模型与联网能力开关（后续可扩展）

## 12.7 各平台大模型 Token 统计

### 功能描述

系统需要对接入的不同大模型平台进行统一的 Token 使用统计，用于成本核算、调用分析、套餐设计和用户侧透明展示。

### 统计目标

- 统计各模型平台的输入 Token、输出 Token、缓存 Token（如平台支持）
- 统计不同模型、不同功能场景下的 Token 消耗
- 统计单用户、单会话、单任务、单时间周期的 Token 使用情况
- 支持内部运营看板与用户侧用量展示

### 支持的平台维度

- OpenAI
- Anthropic
- Google Gemini
- 阿里通义
- 智谱 GLM
- DeepSeek
- Moonshot
- 其他兼容 OpenAI API 的模型服务商

### 核心统计维度

- 平台名称
- 模型名称
- 用户 ID
- 会话 ID
- 任务 ID
- 功能模块
- 请求时间
- 输入 Token
- 输出 Token
- 总 Token
- 命中缓存 Token
- 请求耗时
- 预估成本
- 实际返回状态

### 功能需求

- 每次模型调用后自动记录 Token 用量
- 支持按平台、模型、用户、功能模块聚合统计
- 支持查看当日、近 7 日、近 30 日 Token 趋势
- 支持统计单次任务的 Token 消耗明细
- 支持区分聊天、总结、翻译、提醒生成、Agent 调度等不同场景
- 当第三方平台未返回标准 Token 字段时，支持本地估算或兼容兜底
- 支持为后续会员套餐、额度包、成本预警提供数据基础

### 用户侧展示建议

- 在设置中心增加“模型用量”页面
- 展示当前周期已使用 Token、预估成本、剩余额度
- 展示不同模型的消耗占比
- 当用量异常增长时提示用户

### 管理侧需求

- 后台支持按平台查看总体消耗
- 支持按用户、模型、功能模块筛选
- 支持导出日报/周报
- 支持配置单用户 Token 限额和预警阈值

### 验收标准

- 95% 以上模型调用可被成功记录 Token 数据
- 聚合统计结果与原始调用日志误差小于 3%
- 至少支持按平台、模型、用户、功能模块四个维度筛选
- 用户侧可查看最近 30 天的 Token 用量趋势

## 13. 用户流程

### 13.1 首次使用流程

1. 用户安装并启动应用
2. 桌宠出现在桌面，进行首次欢迎
3. 引导用户设置昵称与基础偏好
4. 引导体验一个快捷能力，如“创建提醒”
5. 完成首次 onboarding

### 13.2 日常使用流程

1. 用户点击桌宠或使用快捷键唤起对话
2. 输入聊天或任务指令
3. 系统识别意图
4. 如需确认则确认后执行
5. 桌宠反馈执行进度与结果
6. 相关信息进入短期上下文或长期记忆

## 14. 交互与体验要求

### 14.1 视觉方向

- 采用轻量、可爱、柔和的金毛犬角色设计
- 默认体积小，不压迫桌面空间
- 动画节奏偏轻，不做高频跳动
- 支持后续换肤和角色扩展
- 默认配色建议为金黄、米白、暖棕，避免高饱和刺激色
- 五官和身体比例偏幼态，但整体气质要温暖可靠，不走过度卡通化路线

### 14.2 交互要求

- 用户 1 次点击可触达主要能力
- 对话面板不应过大，不遮挡主工作区
- 主动打扰必须可配、可关闭
- 执行型动作必须可感知、可取消、可追溯

### 14.3 人设建议

- 物种设定：金毛犬
- 形象关键词：温暖、忠诚、聪明、贴心、亲近、克制
- 说话风格：简洁、温暖、有陪伴感，像一只懂事的金毛犬伙伴
- 行为风格：喜欢陪伴、乐于响应、偶尔撒娇，但不过度打扰
- 行为原则：不装懂、不自作主张、不频繁刷存在感
- 情绪表达映射：
  开心时摇尾巴、完成任务时轻快小跑、思考时歪头、等待时趴下、夜间进入睡觉状态

## 15. 技术方案建议

### 15.1 当前技术架构

桌面端当前确定采用以下组合：

- 桌面容器：Tauri
- 前端 UI：React + TypeScript
- 动画层：Lottie
- 本地任务调度与系统能力：TypeScript + Rust 本地能力模块
- AI 能力层：统一模型网关 + 本地工具调用编排
- 数据存储：SQLite

### 15.2 系统分层

1. 表现层：桌宠渲染、面板 UI、动画反馈
2. 交互层：输入处理、快捷操作、消息展示
3. Agent 层：意图识别、记忆检索、工具调度
4. 模型网关层：模型路由、平台适配、Token 统计、成本计算
5. 工具层：提醒、应用打开、剪贴板处理、本地调用
6. 数据层：设置、记忆、任务、日志、模型调用记录

### 15.3 Agent 能力边界

MVP 只开放低风险、强确定性的工具能力：

- 读剪贴板
- 创建本地提醒
- 打开应用/网页/文件夹
- 本地文本处理

暂不开放：

- 自动发送消息
- 自动删除/修改系统文件
- 自动支付/下单
- 不经确认的外部操作

## 16. 非功能需求

### 16.1 性能要求

- 桌宠常驻状态 CPU 占用低
- 内存占用可控，避免影响办公体验
- 动画与交互保持流畅

建议指标：

- 常驻空闲 CPU 平均占用 < 5%
- 常驻内存占用尽量控制在轻量桌面应用可接受范围内，并持续以 Tauri 方案优化

### 16.2 稳定性要求

- 核心功能异常不导致应用整体崩溃
- 工具执行失败可恢复
- 本地数据异常有兜底策略

### 16.3 隐私与安全

- 敏感权限采用显式授权
- 用户记忆可导出、删除
- 对话与执行日志默认本地保存
- 联网能力需透明告知

## 17. 风险与应对

### 风险 1：产品只“可爱”但“不好用”

应对：

- MVP 优先验证高频任务能力
- 设计关键任务漏斗数据

### 风险 2：主动打扰导致用户反感

应对：

- 默认低频提醒
- 提供免打扰和主动度调节

### 风险 3：Agent 能力不稳定影响信任

应对：

- 收敛到低风险、强规则任务
- 所有执行动作明确确认

### 风险 4：桌面性能消耗过高

应对：

- 优化动画资源与渲染频率
- 首版控制角色复杂度

## 21. 开放问题

- 首发用户更偏“效率工具”还是“情感陪伴”？
- 金毛犬桌宠形象在 Lottie 方案下的动作复杂度与资源规格如何定义？
- MVP 是否需要语音输入？
- 是否首版就接入联网搜索？
- 商业化路径优先订阅、角色皮肤，还是高级 Agent 能力？

## 22. 结论

Agent 桌宠的关键，不在于把桌宠做得更像宠物，而在于把 AI 助手做得更像一个值得长期留在桌面的伙伴。MVP 阶段需要严格控制边界，围绕“陪伴感、低打扰、高频轻任务”做深做透，再逐步扩展更复杂的 Agent 能力和个性成长体系。

如果产品在首版中能同时做到“愿意留着”和“确实有用”，就具备继续演进为桌面 AI 入口的基础。
