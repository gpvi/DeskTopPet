## 状态规则

任务状态只能使用：`todo`、`in_progress`、`blocked`、`review`、`done`。

- `todo`：未开始，依赖未满足也可保持该状态。
- `in_progress`：已有 Agent 正在处理，必须写明 owner。
- `blocked`：被环境、依赖、决策或上游缺陷阻塞，必须写明阻塞原因。
- `review`：实现已完成，等待测试、GUI 验收或代码审查。
- `done`：代码、测试、文档、验收证据均闭环。

## 多 Agent 协作原则

- 以 `05-多Agent任务清单.md` 为唯一任务入口，不再维护分散任务表。
- 每个 Agent 领取任务前先检查 `depends_on` 和 `status`。
- 并行任务必须写明 `parallel_group`，避免多个 Agent 修改同一文件区域。
- 状态变更要同步更新任务行的 `状态`、`owner`、`最近更新` 和 `交付物`。
- 涉及架构边界、权限、安全、持久化的变更必须同步更新对应文档。
- 涉及技术栈替换、依赖升级、运行目标、数据层和安全边界的变更必须先更新 `08-技术选型挑战与优化.md`。
